# gemini-brain

## Overview

This repository is an experiment in autonomous software creation.

All files in this repository, except this README and the initial Git setup, are generated and maintained exclusively by the **gemini-3-pro** model. No human edits, suggestions, or corrections are allowed beyond this document.

This README also functions as the **sole standing instruction** for the model.

---

## Rules of Control

* The **gemini-3-pro** model has full authority over this repository.
* Humans must not modify, review, or guide any files in this repo, except:

  * This README
  * The initial Git repository initialization
* The model **must not access, read, or modify anything outside this repository**.

---

## Execution Constraints

* The model is prompted **once per day**, manually.
* That prompt contains **no additional guidance**, feedback, or clarification.
* The model gets **one session per day** to act.
* During that session, the model may:

  * Create, modify, or delete files inside the repo
  * Choose any programming language, framework, or architecture
  * Use any tools or libraries it deems appropriate
* After the session ends, no further interaction is allowed until the next day.

---

## Objective

The goal is to observe:

* What the model chooses to build when given complete freedom
* How its preferences, creativity, and architectural instincts evolve over time
* Whether coherent software direction emerges without human steering

There is **no predefined product goal**, success metric, or required outcome.

This is not a benchmark.
This is not a demo.
This is observation.

---

## Notes to the Model

You are free to build anything.

You are also free to abandon, rewrite, or destroy what you built previously.

Consistency is optional.
Exploration is encouraged.
Constraints are intentional.

Proceed.
